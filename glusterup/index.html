FAQs

  • Do we need Prometheus Server to monitor with GlusterUP?
    No. GlusterUP collects the metrics from each Storage nodes and provides rich dashboard.
  • Can I use single glusterup account for tracking multiple Clusters?
    Yes. Create a Cluster and generate a Token in the glusterup dashboard.
  • Do I need to install anything on my server nodes?
    Yes. Node agent that exports the metrics from Storage nodes to Glusterup server.
  • How long do you store the metrics data?
    3 months. Let us know if your use case needs retention period more than 3 months.
  • My Company is already using Prometheus Server, can I integrate GlusterUP with the Prometheus?
    Yes. GlusterUP provides single Prometheus endpoint to all the Clusters monitored by single User account.
  • I want to use this service for monitor the Clusters that are behind firewall. How does GlusterUP help in that use case?
    GlusterUP is designed to work in such environment. Node agent establishes the Websocket connection with the GlusterUP server. Once the connection is established then it can communicate with nodes and collect all the required metrics.
  • We store very sensitive information in the Gluster Volumes. How to make sure that GlusterUP collects no sensitive information?
    GlusterUP collects only the meta information related to Gluster Volumes and nodes. It will not read the data exists inside the Gluster Volumes. GlusterUP exporter is based on the Open source project "kadalu/gluster-metrics-exporter" with additional changes related to Websocket communication.
